.json, .js, .html

I projekter vil der blive benyttet mange forksllige fil-typer som vil sammen danne
f.eks. en hjemmeside med design og funktioner eller en server som kan kommunikerer
med en database som kan gemme og retunerer data til brugeren.

Vores .json filer kan indeholde json objecter. Det kunne være en liste med
biltyper eller det kunne f.eks. være vores package.json fil som indeholder vores
Dependencies, metadata og scripts. I denne applikations primære JavaScript fil kaldet index.js er der defineret en liste kaldet "cars".
Listen cars kan tilgås via localhost/cars hvor de bliver returneret som JSON. Det er også muuligt at tilgå de enkelte "cars" via localhost/cars/3 hvor
tallet (3) indikerer hvilken en id som du ønsker at tilgå. Dette kan udskiftes med et vilkåreligt tal imellem 1-6.
I index.js filen vil man kunne finde nogle variabler med typerne const og let. Const benyttes for at definerer at
variablen er konstant, altså ikke kan ændres efterfølgende. Let bliver brugt til at "scope" variablen så den kun
er tilgængelig hvor den er defineret. I tidligere JavaScript benyttede man var, men dette bliver ikke brugt længere
hvis det kan undås.
Der er også muligheden for at ændre og tilføje til listen, men det vil blive gennemgået i postman afsnittet.

Vores .js filer er vores JavaScript filer. Disse filer indeholder vores javascripts metoder
Det er disse filer som vil håndterer at retunerer de html side som brugen anmoder
via GET eller gemmer data som er parsed via POST.

Så har vi vores .html filer. Disse vil indeholde vores hjemmeside elementer. For at designe
vores hjemmeside ville der være mulighed for at oprette .css filer og så linke til dem
fra vores html filer. Men jeg har valgt at benytte mig af det online css bibliotek kaldet bulma
men det vil jeg skrive nærmere om lidt senere.
