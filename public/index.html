Introduktion

Dette er forsiden af min mandatory opgave til NodeJs valgfaget for 4 semester. 2020 på KEA.
Opgaven består af et NodeJs webserverproject, herunder diverse frameworks som er defineret i package.json filen.
- Men dette vil blive beskrevet nærmere senere. Det vil være muligt at navigerer imellem diverse HTML sider via intuitive links eller via brug af url.
Dette er grundet at programmet er bygget på baggrund af REST API'en. REST står for Representational state transfer
og er en standraliseret måde at behandle efterspøgsler på webservices. REST API benytter sig af JSON.
Siden at REST er standraliseret så betyder det at man ved hvordan den givende API virker unden at kende
meget til den, hvilket giver mulighed for at benytte den uden at læse om den.
Nedenfor kan man se opdelingen imllem Collection resource og Member resource hvor collection døkker over
en gruppe af elementer hvor memener dækker over de enkelte elementer i en collection.
HTML siderne er "udsmykket" via css fra bulma biblioteket som er linket i toppen af hver HTML fil.

Node package manager

Node package manager også kaldet NPM er som navnet beskriver en pakke-manager designet for JavaScript.
NPM bliver primært tilgået fra en computers terminal hvor diverse kommandoer kan udføres fra som f.eks.
- install, update, uninstall og ls for at nævne et par stykker.
NPM bliver installeret samtidig med Node.Js.

Express

Express er et Node.Js framework som fokuserer på at være et "letvægt" og "robust" værktøj
til at udvikle web og mobil applikationer via HTTP. Efter installation bliver en folder kaldet node_modules
tilføjet til projektet hvori at all Express funktioner vil kunne findes. Express bliver installeret via terminalen. Her er to muligheder. 1. Første mulighed er at man manuelt opretter en gyldig package.json fil (Husk "{}").
Man definerer så de dependencies som man ønsker at installerer, i dette tilfælde
express. Dette gøres på denne måde.

"dependencies": {
"express": "^4.17.1"
}

Herefter benyter man sig af terminalen og skriver "npm install".
npm vil så kigge i package.json filen og installerer de dependencies som er defineret.

2. Anden mulighed er at man opretter en gyldig package fil som mulighed 1.
Herefter benytter man sig af terminalen og skriver "npm install express".

3. Tredje mulighed er at man benytter sig af "npm init".
npm vil så gennemgå et setup hvor at der i sidste ende vil være oprettet en pakage.json fil
Denne fil vil så indeholde diverse metadata som navn, licens osv. Men det vil være muligt at
installerer express via mulighed 1 eller 2 herfra.
Man kan også skrive "npm install `pakkenavn`" for at installerer f.eks. express
med det samme.

cross-env

Når man kører et webprogram som f.eks. et af vores projekter, så kan der forekomme
episoder hvor man gerne vil definerer environment variabler som f.eks. hvilket
port-nr. som serveren skal kører på. Dette er rimelig simpelt hvor mac og linux systemer
men det skaber ofte problemer, defor har man udviklet cross-env som løser dette. Cross env installerers på samme måde som express her skrives bare "npm install --save-dev cross-env"
I package.json filen vil så blive tilføjet følgende hvilket bekræfter at cross-env er blevet installeret

"devDependencies": {
"cross-env": "^7.0.2"
}

For at benytte sig af cross-env så skal der defineres to scripts i package.json filen.

"scripts": {
"start": "cross-env PORT=3000 node app.js",
"start-dev": "cross-env PORT=80 nodemon app.js"
}

På denne måde kan vi skrive "npm run start" eller "npm run start-dev" og vil
herfter kører den tilknyttede kommando.
